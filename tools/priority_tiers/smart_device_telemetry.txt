# Smart device / IoT telemetry endpoints — OPT-IN tier.
#
# Blocking these can break or degrade the device. Use only when you
# specifically want to stop the device phoning home. Documented per
# CLAUDE.md feature 3 (Smart Devices Bypass Pi-hole, XDA 2026 report)
# and the broader IoT-tracking literature.
#
# This file is intentionally NOT included in the default
# `[blocklist] sources` config. Add manually:
#   [blocklist]
#   sources = ["blocklists/tier1.txt",
#              "tools/priority_tiers/smart_device_telemetry.txt"]
#
# Sources:
#   - "Smart Devices Bypass Pi-hole" (XDA 2026)
#   - Apthorpe et al. "Spying on the Smart Home" (PETS 2017)
#   - Mozilla Privacy Not Included reports (2023, 2024)

# --- Amazon Alexa / Echo telemetry ---
# Blocking these stops the device from phoning home periodically;
# the assistant still works because voice commands route via a
# different domain (avs-alexa-*.amazon.com — left alone).
0.0.0.0 device-metrics-us.amazon.com
0.0.0.0 device-metrics-us-2.amazon.com
0.0.0.0 pa.csiservice.amazon.com
0.0.0.0 dcape-na.amazon.com

# --- Apple device telemetry (judgement: blocking some breaks features) ---
# DO NOT block: gsp10-ssl.apple.com (security/keychain), captive.apple.com
# (network captive detection — needed on public Wi-Fi).
0.0.0.0 metrics.icloud.com
0.0.0.0 metrics.apple.com

# --- Google Home / Nest / Android telemetry ---
# DO NOT block: mtalk.google.com (Cloud Messaging — used for ALL
# Android push) or *.googleapis.com (Maps, Drive, Auth all break).
0.0.0.0 firebaselogging.googleapis.com
0.0.0.0 firebaseio.com
0.0.0.0 crashlytics.com
0.0.0.0 app-measurement.com
0.0.0.0 firebase-settings.crashlytics.com
