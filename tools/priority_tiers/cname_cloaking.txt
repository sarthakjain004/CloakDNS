# CNAME-cloaking trackers — hand-curated, research-cited.
#
# These domains are documented as the canonical CNAME-cloaking
# providers: a first-party-looking subdomain (e.g.
# `metrics.example.com`) is set up as a CNAME pointing at one of these
# domains, so the browser treats the request as same-origin and ITP /
# extension cookie heuristics are bypassed. The cloakdns CNAME
# uncloaker walks the chain and matches against this list.
#
# Sources:
#   - Dao et al. "CNAME Cloaking and Tracking" (IEEE TNSM 2021)
#   - Vekaria et al. "SoK: Web Tracking" (arXiv 2025) §3.4
#   - NextDNS cname-cloaking-blocklist (we also fetch this dynamically
#     via tools/blocklist_sources.toml; this file is the offline
#     fallback for the well-known providers).
#
# Notes:
#   - Some of these are listed by their CNAME *target* domain (the
#     hidden one), not the first-party-looking name. The uncloaker
#     reads the CNAME chain hop-by-hop and matches each.
#   - Several are also direct cookie-syncing partners and would be
#     caught by tier1.txt / cookie_syncing.txt. We list them here too
#     for clarity and because the suffix-match in cloakdns is O(1).
