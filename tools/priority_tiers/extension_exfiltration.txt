# CloakDNS priority tier — browser-extension data-exfiltration endpoints
#
# Arcanum (2024) found 3,028 Chrome extensions (2.68% of the store) that
# automatically exfiltrate user data via outbound web requests, 65 of
# them over plain HTTP. The dominant exfil channels are: ad-tech /
# analytics endpoints injected into the request stream, telemetry
# servers operated by the extension vendor, and consumer-VPN backends
# that capture browsing history.
#
# This tier focuses on **the receiving endpoints** so blocking them at
# the DNS layer kills the exfiltration leg regardless of which
# extension is sending. Browser extensions can't see system DNS, so
# they cannot reach a CloakDNS-listed endpoint without going through
# CloakDNS (assuming the OS resolver is set to CloakDNS).
#
# Format: <IP> <hostname>  (one rule per line; suffix-matched).
#
# Research provenance per group is in the inline comments below.

# ---- Consumer-VPN extensions that exfiltrate browsing history ----------
# Touch VPN (~8M Chrome users), Hola VPN (~6M), Hideman, and similar
# free-VPN extensions have been repeatedly documented (Arcanum 2024,
# CSIRO 2018, Citizen Lab 2020) routing the user's clickstream through
# their own logging servers in addition to providing the VPN.

0.0.0.0 touchvpn.net
0.0.0.0 hola.org
0.0.0.0 holavpn.net
0.0.0.0 holacdn.com
0.0.0.0 luminati.io
0.0.0.0 brightdata.com
0.0.0.0 hideman.net
0.0.0.0 betternet.co

# ---- Telemetry / "extension health" hubs --------------------------------
# Generic SaaS analytics SDKs (Amplitude, Mixpanel, AdRoll, MaxCDN
# stats) ARE bundled by exfiltrating extensions per Arcanum 2024, but
# they're also legitimately used by many normal websites. Blocking
# them at the DNS layer overblocks. Users who want maximum protection
# against extension exfiltration AND don't mind breaking analytics on
# any site can extend this tier with:
#
#     a.amplitude.com api.mixpanel.com cdn.mxpnl.com d.adroll.com
#     stats.maxcdn.com
#
# We deliberately leave them out of the default tier to keep this list
# specifically about extension-vendor-controlled exfil endpoints.
