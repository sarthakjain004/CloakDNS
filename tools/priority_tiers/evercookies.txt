# CloakDNS priority tier — evercookie / respawning identity infrastructure
#
# Evercookies persist user identity across sessions by writing the same
# value into many different storage layers (cookies, LocalStorage, IndexedDB,
# Flash LSOs, ETag, HSTS, canvas-cached state, ...). After the user clears
# any subset, the cookie respawns from a survivor. Soltani et al. and
# Kamkar's original demo showed the attack; Acar et al. (CCS 2014) and
# Englehardt & Narayanan (CCS 2016) documented the production deployments.
# Modern device-fingerprinting / identity-resolution vendors play the same
# functional role at scale today (you fingerprint the device, you can
# rebuild the cookie from the back end).
#
# Format: <IP> <hostname>  (one rule per line; suffix-matched by CloakDNS)
#
# Research provenance per group is in the inline comments below.
